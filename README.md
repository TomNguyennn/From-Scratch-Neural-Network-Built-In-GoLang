# 🧠 From-Scratch Neural Network in GoLang

[](https://golang.org/)
[](https://go.dev/)

## 📌 Overview

This repository contains a complete, ground-up implementation of a Neural Network written entirely in standard Go (Golang). By bypassing high-level machine learning frameworks like TensorFlow or PyTorch, this project aims to expose the "black box" of AI. It handles everything from raw CSV ingestion and data splitting to forward propagation, backpropagation, and weight updates using standard math and native Go data structures.

## 📂 Repository Structure

The codebase is organized into modular components highlighting both the data engineering and algorithmic sides of Machine Learning:

  * **`neural_network.go`**: The core computational engine. This file defines the neural network architecture (layers, neurons, weights, biases), activation functions, forward pass logic, and the backpropagation algorithm.
  * **`train_test_split.go`**: A data-preprocessing utility. It reads raw datasets, shuffles the data, and partitions it into dedicated sets to ensure the network is trained and evaluated rigorously without data leakage.
  * **`Iris.csv` / `archive.zip`**: The famous Iris botanical dataset, a staple in pattern recognition, used here as the benchmark for multi-class classification.
  * **`train.csv`, `validation.csv`, `test.csv`**: The output artifacts generated by the split script, ingested by the neural network during its respective training, tuning, and evaluation phases.
  * **`go.mod` & `go.sum`**: Go module files for dependency and package management.

## 🔬 Core Concepts & Knowledge Base

### 1\. Data Processing & The Split (`train_test_split.go`)

Before a network can learn, data must be staged correctly. This module parses `Iris.csv` using Go's standard `encoding/csv` package.
It splits the dataset into three vital categories:

  * **Training Set:** The data the network actually learns from by calculating gradients and updating weights.
  * **Validation Set:** Used periodically during training to tune hyperparameters and prevent **overfitting** (memorizing the training data instead of generalizing).
  * **Test Set:** Completely unseen data used at the very end to evaluate the final accuracy of the model.

### 2\. Forward Propagation (`neural_network.go`)

This is the process of pushing data through the network to get a prediction.

  * **Linear Algebra via Go Slices:** Since we are building from scratch, matrix dot-products are computed using 2D Go slices (`[][]float64`).
  * **Weighted Sum:** Each node multiplies incoming inputs by a weight, sums them up, and adds a bias ($z = Wx + b$).
  * **Activation Functions:** The raw sum is passed through an activation function (like Sigmoid or ReLU) built using the `math` package. This introduces *non-linearity*, allowing the network to learn complex, non-linear boundaries.

### 3\. Backpropagation & Gradient Descent

The true "learning" phase. Once the forward pass yields a prediction, the network calculates the error against the actual label.

  * **Loss Calculation:** Determines how far off the prediction was.
  * **Chain Rule of Calculus:** The network computes the gradient of the loss function backwards through the network layers.
  * **Weight Updating (Gradient Descent):** Go iteratively adjusts the weights and biases in the opposite direction of the gradient to minimize the overall error on the next pass.

### 4\. Why Build This in Go?

While Python dominates ML, Go offers distinct advantages for backend inference:

  * **Type Safety & Compiled Speed:** Go is statically typed and compiled directly to machine code, resulting in highly performant mathematical loops compared to interpreted languages.
  * **Concurrency:** Go's Goroutines allow for parallelizing data batches during training or simultaneously calculating gradients without heavy overhead.
  * **No External C-Bindings:** Running a purely native network means simplified deployment without requiring massive external libraries (like `libtorch`).

## 🚀 Getting Started

### Prerequisites

Make sure you have [Go](https://go.dev/doc/install) installed (version 1.16+ recommended).

### 1\. Prepare the Data

First, run the data pipeline to shuffle the `Iris.csv` dataset and generate your splits:

```bash
go run train_test_split.go
```

*This will generate `train.csv`, `validation.csv`, and `test.csv` in your working directory.*

### 2\. Train the Network

Initialize the network and start the training loop:

```bash
go run neural_network.go
```

*You will see the network outputting its loss dropping over the epochs, eventually concluding with a final accuracy score on the `test.csv` dataset.*

# From-Scratch Iris Classifier in Go

This project builds a small neural-network classifier for the Iris dataset in pure Go. It now covers the full workflow:

- split and encode the raw dataset
- train a saved model artifact
- predict one sample or many samples from CSV
- use goroutines for preprocessing and batch inference

## Project Layout

- `train_test_split.go`
  - reads `Iris.csv`
  - one-hot encodes the species label
  - uses a worker-pool pipeline to parse rows concurrently
  - writes `train.csv`, `validation.csv`, and `test.csv`
- `neural_network.go`
  - loads the split datasets
  - fits feature normalization from the training set
  - trains the neural network
  - evaluates train, validation, and test metrics
  - saves the trained model to `model.json`
- `predict.go`
  - runs single-sample inference from CLI flags
  - runs batch prediction from a CSV file
  - uses goroutines for concurrent batch inference
- `internal/irisnn`
  - reusable model, scaler, dataset, and model I/O logic

## Requirements

- Go installed locally
- the Iris dataset present as `Iris.csv`

If your environment needs the explicit local toolchain, use:

```bash
GOTOOLCHAIN=local GO111MODULE=on GOCACHE="$(pwd)/.gocache" /usr/local/go/bin/go <command>
```

Otherwise, plain `go run` and `go test` commands are enough.

## 1. Split the Dataset

Generate stratified train, validation, and test splits:

```bash
go run train_test_split.go --workers 4
```

Useful flags:

- `--input`
- `--train`
- `--validation`
- `--test`
- `--train-ratio`
- `--validation-ratio`
- `--test-ratio`
- `--seed`
- `--workers`

The preprocessing pipeline uses this concurrency pattern:

- one feeder goroutine sends CSV rows into a `jobs` channel
- multiple worker goroutines validate and encode rows
- one collector receives `results` and owns the output slice

That design keeps the splitting logic and file writes single-threaded while avoiding write races.

## 2. Train the Model

Train the neural network and save the model artifact:

```bash
go run neural_network.go --model-out model.json --epochs 100 --learning-rate 0.15 --hidden-neurons 10 --seed 42
```

Outputs:

- training loss printed across epochs
- train, validation, and test loss/accuracy
- saved model artifact in `model.json`

The saved artifact includes:

- weights and biases
- network configuration
- scaler mean/std
- label names
- evaluation metrics

## 3. Predict a Single Flower

Run inference for one flower measurement:

```bash
go run predict.go --sepal-length 5.1 --sepal-width 3.5 --petal-length 1.4 --petal-width 0.2
```

Output:

- predicted species
- class probabilities for all three labels

## 4. Run Batch Prediction

Predict a full CSV with concurrent workers:

```bash
go run predict.go --batch-input test.csv --batch-output predictions.csv --workers 4 --benchmark-repeats 3
```

Batch mode:

- reads rows from the input CSV
- normalizes features with the saved scaler
- predicts labels concurrently
- preserves original row order in the output file
- prints timing for each benchmark repeat

The output CSV appends:

- `PredictedSpecies`
- `ProbSetosa`
- `ProbVersicolor`
- `ProbVirginica`

## Testing

Run the test suite:

```bash
go test ./...
```

Run the race detector on the concurrent prediction path:

```bash
go run -race predict.go --batch-input test.csv --batch-output predictions.csv --workers 4 --benchmark-repeats 1
```

Run the race detector on concurrent dataset preprocessing:

```bash
go run -race train_test_split.go --workers 4
```

Current automated tests cover:

- feature extraction from both plain and labeled CSV rows
- batch prediction output ordering under worker concurrency
- model save/load round-trip behavior
